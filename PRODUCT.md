# PRODUCT.md — AI Scientist (Autosearch)

## Users

- **Researchers** (ML/AI PhD students, postdocs, lab leads) who want to automate the research cycle: from idea generation to paper writing and peer review. They work long hours, often at night, and need a tool that accelerates their workflow without babysitting.
- **R&D Engineers** in industry exploring automated scientific discovery. They want a dashboard that shows pipeline progress at a glance and lets them intervene when needed.

## Product Purpose

An autonomous research agent that reproduces the Sakana AI Scientist algorithm. Users launch a pipeline, and the system generates ideas, runs experiments, writes papers, and submits them to automated peer review, looping back on rejection. The interface makes this multi-day process visible, navigable, and controllable.

## Brand

**Autosearch** — the name suggests tireless, systematic exploration. The brand is intellectual rigor, not flash. Think laboratory instrument, not startup landing page. Quiet confidence. The interface should feel like a well-designed research tool, not a marketing demo.

## Tone

- Precise, calm, scientific. No hype, no exclamation marks in UI copy.
- Status labels use neutral language: "In Progress", not "🚀 Running!"
- Errors are explained, not just flagged.

## Anti-references

- Not a SaaS marketing dashboard with hero metrics and gradient cards.
- Not a chat interface. This is a pipeline orchestration tool.
- Not a generic admin template with sidebar + data tables.

## Strategic Principles

1. **Transparency over magic.** Show what the agent is doing, what it decided, and why. Every step of the pipeline is visible.
2. **Intervention points.** The user can pause, redirect, or override at any phase. Autonomy with oversight.
3. **Temporal awareness.** Research pipelines run for hours. The UI must communicate progress and ETA, not just status.
4. **Density where needed.** Paper reviews, experiment logs, and literature search results need information density, not whitespace.
